# Research Notes (Public UX scan)

Date: 2026-03-07
Method: Public homepage flow observation and public pattern analysis.

## Observed Core UX Patterns
1. Top-level tabs: Homes / Experiences / Services.
2. Universal search model: Where + When + Who.
3. Personalized location-based listing rails on homepage.
4. Card-level trust signals:
   - Guest favorite label
   - Rating visible on card
   - Total trip price visible
5. Price transparency messaging surfaced prominently.
6. Strong host CTA in header (Become a host).

## Product Implications for StayHia
- Keep search primitive dead simple and globally consistent.
- Keep price breakdown explicit early to improve conversion trust.
- Build trust labels as first-class entities (not hardcoded tags).
- Treat host onboarding as a growth funnel, not just settings forms.

## Guardrails
- Do not copy proprietary assets/code/text verbatim.
- Build independent implementation using public product patterns.
- Ensure legal and compliance localization by region.
